# Prompt Patterns – Tekrar Kullanılabilir Prompt Tasarım Kalıpları

Bu bölümde, tekrar eden problem tipleri için kullanılan
standart prompt tasarım kalıpları (patterns) tanıtılır.

Amaç:

tek seferlik iyi prompt yazmak değil,
benzer problemler için tekrar kullanılabilir tasarım şablonları oluşturmaktır.

---

## Prompt pattern nedir?

Prompt pattern, belirli bir problemi çözmek için
kullanılan yapılandırılmış prompt tasarım yaklaşımıdır.

Bir pattern:

- hangi sırayla talimat verileceğini
- modelden hangi ara çıktının isteneceğini
- çıktıların nasıl organize edileceğini

tanımlar.

---

## Bu bölümde hangi pattern’ler var?

Bu repoda üç temel pattern ele alınmaktadır:

- Few-shot prompting
- Chain-of-thought prompting
- Self-critique prompting

Bu pattern’ler:

- yazılım geliştirme
- veri / yapay zeka çalışmaları
- analiz ve raporlama

senaryolarında doğrudan uygulanabilir.

---

## Pattern seçimi neden önemlidir?

Aynı problemi farklı pattern’lerle çözmek mümkündür.
Ancak yanlış pattern seçimi:

- gereksiz uzun çıktılar
- hatalı muhakeme
- kararsız cevaplar

üretebilir.

Bu nedenle pattern seçimi,
prompt tasarımının önemli bir parçasıdır.
